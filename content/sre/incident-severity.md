# 장애 심각도 등급 정의

## 1. 심각도 등급 체계

심각도(Severity)를 명확히 정의해야
적절한 대응 속도와 자원을 배치할 수 있다.

| 등급 | 이름 | 영향 범위 | 대응 시간 |
|------|------|---------|---------|
| SEV-1 | Critical | 서비스 전체 중단 | 즉시 (15분 이내) |
| SEV-2 | High | 핵심 기능 장애 | 30분 이내 |
| SEV-3 | Medium | 비핵심 기능 저하 | 4시간 이내 |
| SEV-4 | Low | 사소한 이슈 | 다음 영업일 |

---

## 2. 등급별 상세 기준

### SEV-1 (Critical)

```
조건:
  □ 전체 서비스 접근 불가
  □ 결제, 로그인 등 핵심 기능 완전 중단
  □ 데이터 손실 발생 중
  □ 보안 침해 진행 중

예시:
  - 데이터베이스 전체 다운
  - CDN 전역 장애
  - 랜섬웨어 감염

대응:
  → 전체 온콜 호출
  → C-Level 즉시 보고
  → 15분마다 상태 업데이트
```

### SEV-2 (High)

```
조건:
  □ 핵심 기능 부분 장애
  □ 일부 사용자 영향 (> 10%)
  □ SLO 위반 중

예시:
  - 결제 실패율 10% 이상
  - 특정 리전 접속 불가
  - API 응답 속도 10배 저하

대응:
  → 담당 온콜 호출
  → 팀 리드 보고
  → 30분마다 업데이트
```

### SEV-3 (Medium)

```
조건:
  □ 비핵심 기능 장애
  □ 소수 사용자 영향 (< 5%)
  □ 우회 방법 존재

예시:
  - 부가 기능 일시 오류
  - 특정 브라우저 렌더링 이슈

대응:
  → 슬랙 알림
  → 당일 처리 목표
```

## 3. 영향 평가 매트릭스

```
영향 범위 × 심각성:

              경미   보통   심각
전체 사용자   SEV-2  SEV-1  SEV-1
일부 사용자   SEV-3  SEV-2  SEV-1
소수 사용자   SEV-4  SEV-3  SEV-2
```

---

## 4. 등급 조정

```
업그레이드 (낮은 등급 → 높은 등급):
  → 새로운 영향 발견 시
  → 빠른 확산 조짐 시

다운그레이드 (높은 → 낮은):
  → 실제 영향이 예상보다 작을 때
  → 우회 방법 적용 완료 후
  → 반드시 팀 전체에 공지
```
