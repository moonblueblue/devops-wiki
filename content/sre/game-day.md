# 모의 장애 훈련 설계 (Game Day)

## 1. 개념

```
Game Day:
  팀 전체가 모여 실제 장애를 모의로 경험하는
  훈련 세션.

목적:
  □ 온콜 프로세스 검증
  □ 팀 대응 역량 강화
  □ 도구/런북 유효성 확인
  □ 약점 사전 발견
```

---

## 2. 게임데이 설계 원칙

```
안전:
  → 중단 버튼을 항상 준비
  → 비즈니스 영향 낮은 시간대 선택
  → 처음엔 스테이징에서 시작

현실적:
  → 실제 발생 가능한 시나리오
  → 실제 도구와 프로세스 사용

학습 중심:
  → 성공/실패보다 배움이 목적
  → 결과로 팀 평가하지 않음
```

---

## 3. 시나리오 예시

### 시나리오 1: DB 페일오버

```
설명: 주 데이터베이스 강제 중단

사전 조건:
  □ DB 복제 구성 확인
  □ 애플리케이션 연결 풀 설정 확인

실험:
  1. 주 DB 인스턴스 중지
  2. 페일오버 소요 시간 측정
  3. 애플리케이션 자동 재연결 확인

성공 기준:
  □ 서비스 중단 < 30초
  □ 데이터 손실 없음
  □ 자동 알림 발송 확인
```

### 시나리오 2: 트래픽 급증

```
설명: 10배 트래픽 주입

사전 조건:
  □ HPA 설정 확인
  □ 부하 테스트 도구 준비

실험:
  1. k6로 점진적 부하 증가
  2. HPA 스케일아웃 동작 확인
  3. 에러율 모니터링

성공 기준:
  □ 에러율 < 1%
  □ HPA 5분 내 스케일아웃
  □ p99 레이턴시 < 2초
```

## 4. 게임데이 진행 순서

```
D-2주: 계획
  □ 시나리오 선택
  □ 참가자 결정
  □ 일정·공지

D-1일: 준비
  □ 메트릭 베이스라인 수집
  □ 도구 동작 확인
  □ 롤백 계획 확인

게임데이 당일:
  □ 킥오프 (목표·규칙 공유)
  □ 시나리오 실행
  □ 실시간 관찰·기록
  □ 복구
  □ 즉각 회고

D+3일: 사후 분석
  □ 포스트모템 작성
  □ 액션 아이템 등록
```
