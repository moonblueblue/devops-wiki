# 서브넷과 CIDR

## 1. IP 주소 구조

IPv4 주소는 32비트이며 점으로 구분된 4개 옥텟으로 표현한다.

```
192.168.1.100
│   │   │ │
│   │   │ └── 호스트 부분
│   │   └──── 네트워크 부분
└───┴──────── 네트워크 부분
```

### 사설 IP 주소 대역 (RFC 1918)

| 대역 | CIDR | 호스트 수 | 용도 |
|------|------|---------|------|
| 10.0.0.0 | /8 | 약 1,677만 | 대규모 기업망, 클라우드 VPC |
| 172.16.0.0 | /12 | 약 104만 | 중간 규모 |
| 192.168.0.0 | /16 | 약 6만 5천 | 소규모, 가정망 |

---

## 2. CIDR 표기법

CIDR(Classless Inter-Domain Routing)은 prefix 길이로
네트워크 크기를 표현한다.

```
192.168.1.0/24
           ↑
           prefix length: 네트워크 부분 비트 수
```

### Prefix별 호스트 수

| CIDR | 서브넷 마스크 | 호스트 수 | 실제 사용 가능 |
|------|------------|---------|-------------|
| /8 | 255.0.0.0 | 16,777,216 | 16,777,214 |
| /16 | 255.255.0.0 | 65,536 | 65,534 |
| /24 | 255.255.255.0 | 256 | 254 |
| /25 | 255.255.255.128 | 128 | 126 |
| /26 | 255.255.255.192 | 64 | 62 |
| /28 | 255.255.255.240 | 16 | 14 |
| /30 | 255.255.255.252 | 4 | 2 |
| /32 | 255.255.255.255 | 1 | 1 (단일 호스트 지정, 예외) |

> 실제 사용 가능 수 = 전체 - 2 (네트워크 주소 + 브로드캐스트 주소)
> /32는 단일 호스트 지정 용도로 네트워크/브로드캐스트 구분 없는 예외 케이스다.

```bash
# 서브넷 계산 도구
ipcalc 192.168.1.0/24
# Network:   192.168.1.0/24
# Broadcast: 192.168.1.255
# HostMin:   192.168.1.1
# HostMax:   192.168.1.254
# Hosts/Net: 254
```

## 3. 서브넷 분할 예시

### /24 → /25 분할 (2개 서브넷)

```
192.168.1.0/24 분할
    ├── 192.168.1.0/25   (0~127, 126개 호스트)
    └── 192.168.1.128/25 (128~255, 126개 호스트)
```

### /24 → /26 분할 (4개 서브넷)

```
192.168.1.0/24 분할
    ├── 192.168.1.0/26   (0~63)
    ├── 192.168.1.64/26  (64~127)
    ├── 192.168.1.128/26 (128~191)
    └── 192.168.1.192/26 (192~255)
```

### 실무 VPC 서브넷 설계 예시

```
VPC: 10.0.0.0/16

  퍼블릭 서브넷 (AZ-A)  : 10.0.1.0/24   (254 호스트)
  퍼블릭 서브넷 (AZ-B)  : 10.0.2.0/24
  프라이빗 서브넷 (AZ-A): 10.0.10.0/24
  프라이빗 서브넷 (AZ-B): 10.0.11.0/24
  DB 서브넷 (AZ-A)      : 10.0.20.0/24
  DB 서브넷 (AZ-B)      : 10.0.21.0/24
```

## 4. 같은 서브넷 판별

두 IP가 같은 서브넷에 있는지 확인하는 방법:

```
IP & 서브넷 마스크 = 네트워크 주소

192.168.1.100 & 255.255.255.0 = 192.168.1.0
192.168.1.200 & 255.255.255.0 = 192.168.1.0
→ 같은 서브넷 ✓

192.168.2.100 & 255.255.255.0 = 192.168.2.0
→ 다른 서브넷 ✗ (라우터 필요)
```

```bash
# 라우팅 테이블로 확인
ip route get 192.168.1.200
# 192.168.1.200 dev eth0 src 192.168.1.100
# → 직접 통신 가능 (같은 L2 세그먼트)
```

## 5. 클라우드별 VPC/서브넷 CIDR 지원 범위

| 클라우드 | VPC 범위 | 서브넷 최소 | 서브넷 최대 |
|---------|---------|-----------|-----------|
| AWS | /16 ~ /28 | /28 (16개, 사용 가능 11개) | /16 |
| GCP | 전역 VPC (서브넷 단위 CIDR) | /29 | /8 |
| Azure | /8 ~ /29 | /29 | /8 |

> 모든 클라우드 서비스는 서브넷당 **예약 IP**를 갖는다.

| 클라우드 | 예약 IP 수 | 예약 내용 |
|---------|-----------|---------|
| AWS | **5개** | 네트워크 주소, 라우터, DNS, 예약, 브로드캐스트 |
| GCP | **4개** | 네트워크 주소, 기본 게이트웨이, 두 번째 끝, 브로드캐스트 |
| Azure | **5개** | 네트워크 주소, 기본 GW, DNS(×2), 브로드캐스트 |

> GCP는 AWS/Azure와 달리 **전역(Global) VPC** 구조이며,
> 서브넷이 리전별로 정의되어 VPC 전체 CIDR 개념이 다르다.

## 6. IPv6 서브넷

IPv6는 128비트 주소를 사용한다.

```
2001:db8::/32      → 조직 할당 블록
2001:db8:1::/48    → 사이트
2001:db8:1:1::/64  → 서브넷 (표준 단위)
```

| 표기 | 설명 |
|------|------|
| `/64` | SLAAC 표준 크기 (RFC 4291). EUI-64 또는 임의 인터페이스 ID 자동 생성 |
| `/48` | 조직 사이트 단위 |
| `/128` | 단일 인터페이스 (/32 IPv4 상당) |
